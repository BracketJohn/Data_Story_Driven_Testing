Docker Network is the new alternative to --links. Links is deprecated. Still not tested with docker-compose and don't know how it works with it

Build all images with mvn clean package -DskipTests docker:build

Then: 

docker network create gdis_network

docker run --name gdis_db -d -p 3306:3306 --network-alias gdis_db --network gdis_network -e MYSQL_DATABASE=gdis_db \
		-e MYSQL_USER=gdis_db_user -e MYSQL_PASSWORD=abcd1234 -e MYSQL_ROOT_PASSWORD=abcd1234 mysql/mysql-server:latest


docker run --name database -d -p 8081:8081 --network gdis_network -e DATABASE_HOST=gdis_db -e DATABASE_PORT=3306 \ 
			-e DATABASE_NAME=gdis_db -e DATABASE_USER=gdis_db_user -e DATABASE_PASSWORD=abcd1234 gdis.db/database


docker run --name exporter -d -p 8082:8082 --network gdis_network -e DATABASE_URL=database -e DATABASE_PORT=8081 gdis.exporter/exporter

NOTICE DATABASE_URL is equal to database, which is the name of the Database Microservice Container

docker run --name importer -d -p 8083:8083 --network gdis_network -e DATABASE_URL=database -e DATABASE_PORT=8081 gdis.importer/importer

Then everything is accessible at localhost:MICROSERVICE_PORT/someUrl
